## 1. 项目用途

CPA Codex Tools 是一个本地浏览器 Web 工具，用于管理 CPA 中已经导入的 Codex 账号。它只包含客户端页面，不包含 CPA 服务端，也不会随仓库保存任何真实账号配置。

主要能力：

- 查看账号分组、状态、优先级、5h / 7d 剩余额度、额度刷新时间和证书过期时间。
- 按筛选结果或勾选账号批量查询额度。
- 批量下载远端账号配置 JSON，作为同步优先级或维护前的备份。
- 按当前列表顺序生成优先级本地草稿，并同步回 CPA。
- 通过 Keeper 页面演练或执行账号维护，包括禁用、刷新、删除和重复证书筛选。
- 通过 OAuth 页面辅助失效 Codex 账号重新登录，支持 Hotmail 验证码读取和可选浏览器扩展批量恢复。

日常推荐路径：

1. 启动 Web 页面。
2. 输入 CPA 管理地址和管理密钥登录。
3. 点击 `加载账号`，确认账号列表正常。
4. 执行查询、备份、优先级或 Keeper 操作。
5. 需要 OAuth 重登时，进入 `OAuth` 页面导入 Hotmail 账号池。
6. 需要迁移本地敏感配置时，在设置面板点击 `导出敏感配置`。

## 2. 使用前准备

### 2.1 必备条件

使用前需要准备：

- 一套可访问的 CPA 服务，且已经导入 Codex 账号配置。
- CPA Management API 地址和管理密钥。
- 支持现代 Web API 的浏览器，推荐 Chrome 或 Edge。
- 本地启动方式二选一：
  - Docker Compose。
  - Node.js 20 或更新版本。

浏览器会直接请求 CPA Management API，因此 CPA 或反向代理需要允许本地 Web 页面跨域访问。常见本地地址形如：

```text
http://localhost:8317/v0/management
```

### 2.2 安全注意事项

- 管理密钥会随每次 CPA Management API 请求发送，使用的请求头为 `Authorization: Bearer <managementKey>` 和 `X-Management-Key: <managementKey>`。
- 只建议在可信本机或可信内网环境中使用本工具。
- 如果 CPA 管理地址使用非 HTTPS 且不是本机地址，页面会提示管理密钥可能明文传输。
- 账号配置下载得到的 JSON 文件可能包含 refresh token 等敏感信息，应按密钥文件对待。
- 浏览器本地配置中的管理密钥会加密保存；Hotmail 密码和 refresh token 当前随 OAuth 配置保存在当前浏览器 profile 中。设置面板导出的敏感配置 JSON 是明文，应按密钥文件保存和传输。
- 删除证书、删除重复授权文件和同步优先级都会改动 CPA 远端数据，执行前建议先下载备份。

## 3. 启动项目

### 3.1 使用 Docker Compose

在仓库根目录运行：

```bash
docker compose up -d web
```

打开：

```text
http://localhost:5173
```

查看服务状态：

```bash
docker compose ps web
```

查看日志：

```bash
docker compose logs -f web
```

停止服务：

```bash
docker compose down
```

Docker Compose 会在 `/runtime/vault-secret` 生成并复用浏览器端加密密钥。该文件由 Docker volume 持久化，服务升级或重启时应保留；如果删除 volume，旧浏览器缓存中的加密管理密钥将无法解密。

注意：

- 普通 `docker compose down` 默认不会删除 volume。
- 不要在保留旧浏览器缓存的情况下执行 `docker compose down -v`，否则运行时加密密钥会被删除。
- 如果要迁移部署环境，建议先在旧环境导出敏感配置，或保留对应 runtime secret volume。
- 默认服务使用 `web_runtime_secret`，开发 profile 使用 `web_dev_runtime_secret`，预览 profile 使用 `web_preview_runtime_secret`。

如果需要接近构建产物的预览模式：

```bash
docker compose --profile preview up -d web-preview
```

预览地址：

```text
http://localhost:9999
```

### 3.2 使用 Node.js

在仓库根目录进入 `web/`：

```bash
cd web
npm install
npm run web:dev
```

默认开发地址：

```text
http://127.0.0.1:5173
```

构建前端：

```bash
npm run web:build
```

开发和构建脚本会自动生成 `runtime-secret.js`。不使用 Docker 时，密钥默认保存在 `web/.runtime/vault-secret`，该目录不会提交到仓库。

如需单独生成或复用运行时密钥：

```bash
npm run runtime:secret
```

生成产物：

- `web/.runtime/vault-secret`：本地运行时密钥。
- `web/public/runtime-secret.js`：开发服务使用的浏览器注入脚本。
- `web/dist/runtime-secret.js`：构建预览使用的浏览器注入脚本。

预览构建产物：

```bash
npm run web:preview
```

端口配置位于 `web/web-server.config.ts`。

## 4. 登录与连接配置

首次打开页面会进入登录页。

操作步骤：

1. 在 `CPA 管理地址` 中填写 CPA Management API 地址。
2. 在 `管理密钥` 中填写 CPA 管理密钥。
3. 按需保留或取消 `记住本次登录`。
4. 点击 `登录`。

`记住本次登录` 默认开启：

- 开启时，CPA 地址和加密后的管理密钥会保存到当前浏览器本地配置，下次打开页面会自动验证并进入控制台。
- 关闭时，只保留本次页面会话中的管理密钥；刷新或重新打开后需要重新输入。

登录成功后，页面会进入控制台。进入控制台后点击 `加载账号` 才会请求并刷新账号列表。

右上角 `退出登录` 会清空本地保存的管理密钥，并回到登录页。

如果需要修改 CPA 地址或管理密钥：

1. 点击左侧 `配置`。
2. 修改 `连接配置`。
3. 点击 `保存配置`。
4. 回到 `额度` 页面点击 `加载账号`。

## 5. 页面布局

左侧导航包含四个页面：

- `额度`：账号额度查询、筛选、排序、下载备份、批量优先级。
- `配置`：CPA 管理地址和管理密钥。
- `Keeper`：账号维护演练、执行维护、选中账号禁用/刷新/删除、重复证书筛选。
- `OAuth`：失效账号重登、Hotmail 验证码、OAuth 批量队列。

右上角提供：

- 设置入口：查询并发、Keeper 策略、本地缓存清理。
- 退出登录。

## 6. 额度页面

![主界面总览](readme/dashboard-overview.png)

### 6.1 加载账号

点击 `加载账号` 会从 CPA 读取账号配置列表。加载成功后，账号会出现在表格中。

页面会展示：

- 邮箱或账号名。
- 优先级。
- 分组，例如 `team`、`plus`、`free`、`pro 5x`、`pro 20x`、`unknown`。
- 状态。
- 5h 和 7d 剩余额度。
- 额度更新时间，也就是下一次刷新时间。
- 证书过期时间。
- 最近查询时间。

### 6.2 状态含义

状态说明：

- `正常`：额度查询成功，当前额度未触发低额度或耗尽判断。
- `偏低`：额度查询成功，但剩余额度较低。
- `耗尽`：额度窗口已耗尽。
- `异常`：额度查询失败或账号不可用。
- `未查`：账号列表已加载，但还没有成功查询额度。

查询失败或超时时，页面会尽量保留上一次成功的额度快照，避免短暂故障清空可用信息。

### 6.3 筛选、搜索和排序

可用筛选方式：

- 顶部分组条：按 `全部`、`Team`、`Plus`、`Free`、`Pro 5x`、`Pro 20x`、`Unknown` 筛选。
- 搜索框：按邮箱搜索。
- 总览卡片：点击正常、偏低、耗尽、异常等统计卡片可按状态筛选。
- 表头排序：邮箱、优先级、分组、状态、5h 额度、7d 额度、额度更新时间、证书过期时间、更新时间都支持排序。

当前筛选和排序会影响后续 `查询选中账号`、`批量设置优先级`、`当前筛选结果生成 OAuth 队列` 等操作。

### 6.4 选择账号

在账号表格第一列勾选账号。表头复选框可以选择当前列表中的全部可见账号。

注意：

- 额度页的勾选状态按当前分组保留。
- Keeper 页有自己的全量勾选集合，用于维护操作。

### 6.5 查询额度

操作步骤：

1. 点击 `加载账号`。
2. 通过分组、状态、搜索或排序找到目标账号。
3. 勾选需要查询的账号。
4. 点击 `查询选中账号 (N)`。

页面会显示查询进度。查询完成后会更新账号状态、5h / 7d 剩余额度、额度刷新时间和最近查询时间。

查询并发数可在右上角设置面板中调整，默认值为 `6`。

### 6.6 下载账号配置备份

按钮行为：

- 未勾选账号时，按钮显示 `下载所有账号`，会下载全部账号配置。
- 已勾选账号时，按钮显示 `下载选中 (N)`，只下载勾选账号配置。

下载由浏览器完成，文件会进入浏览器默认下载目录。本项目不需要配置本地备份路径。

建议在以下操作前先下载全量备份：

- 同步优先级到远端。
- 执行 Keeper 维护。
- 删除账号证书或重复授权文件。

## 7. 批量设置优先级

![优先级设置](readme/priority-batch-panel.png)

批量优先级入口在额度页工具条：`批量设置优先级`。

### 7.1 推荐流程

1. 在额度页加载账号。
2. 使用分组、状态、搜索框收敛本次要调整的账号范围。
3. 点击表头排序，决定分组内部账号顺序。
4. 点击 `批量设置优先级`。
5. 勾选要调整的分组。
6. 用左右按钮调整分组顺序。
7. 为每个分组设置最小和最大优先级。
8. 点击 `生成本地草稿`。
9. 在账号列表中检查 `未同步` 标记和优先级变化。
10. 点击 `同步到远端`。

### 7.2 生成规则

- 只对当前列表中的账号生成草稿，当前筛选、搜索和排序都会参与计算。
- 只会改动弹层里勾选的分组。
- 未勾选分组、当前筛选之外的账号和已有本地草稿会保留。
- 分组顺序越靠前，默认优先级区间越高。
- 同一分组内部按当前账号列表顺序从高到低生成优先级。
- 如果账号数多于优先级档位数，会按档位平均分配。
- 如果账号数少于档位数，只使用最高的前几个档位。
- 如果最小值和最大值相同，该分组所有账号会得到相同优先级。

### 7.3 草稿与同步

生成优先级后，页面只产生本地草稿，不会立即写入 CPA。带 `未同步` 标记的账号表示本地草稿与远端值不同。

可用操作：

- `清除本地草稿`：放弃当前页面里的本地优先级草稿。
- `同步到远端`：把草稿写回 CPA。

首次同步前，如果本轮草稿生成后没有做过全量下载备份，页面会弹出确认框。推荐选择先下载备份再同步。

本地草稿只存在当前页面内存中，刷新页面后会消失。已经同步到 CPA 的远端优先级不会受影响。

## 8. 设置面板

![设置与缓存](readme/settings-panel.png)

点击右上角设置入口可打开 `查询设置`。

可配置项：

- `并发数`：额度查询并发数，默认 `6`。
- `界面 / 页面模式`：可选择跟随系统、亮色或暗色。
- `界面 / 语言`：可在中文和 English 之间切换。
- `Keeper 策略 / 禁用阈值`：Keeper 判断额度阈值，范围 `0 - 100`。
- `Keeper 策略 / 过期阈值天数`：证书临期判断天数，默认 `3`。
- `Keeper 策略 / 维护并发数`：Keeper 维护并发数，默认 `6`。
- `维护时自动刷新临期证书`：执行维护时是否自动刷新临期证书。
- `导出敏感配置`：下载明文 JSON，包含 CPA 地址、管理密钥、Hotmail 邮箱、密码、Client ID 和 refresh token。
- `清空本地缓存`：清除当前浏览器保存的 CPA 地址、管理密钥、账号列表缓存、额度快照和 Web 端配置。

清空本地缓存后需要重新登录。该操作也会清除浏览器 fallback 加密密钥；如果没有部署侧的运行时密钥，旧缓存里的加密敏感配置将无法再解密。

### 8.1 导出敏感配置

导出步骤：

1. 点击右上角设置入口。
2. 在 `本地数据` 区域点击 `导出敏感配置`。
3. 阅读确认弹窗，确认后浏览器会下载 JSON 文件。

文件名格式：

```text
cpa-codex-sensitive-export-YYYYMMDD-HHmmss.json
```

导出内容结构：

```json
{
  "schema_version": 1,
  "exported_at": "2026-05-11T00:00:00.000Z",
  "cpaBaseUrl": "http://localhost:8317/v0/management",
  "managementKey": "明文管理密钥",
  "oauthSettings": {
    "hotmailAccounts": [
      {
        "email": "alice@hotmail.com",
        "password": "明文 Hotmail 密码",
        "clientId": "client-id",
        "refreshToken": "Hotmail refresh token"
      }
    ]
  }
}
```

导出文件是明文，只适合放在可信位置做人工备份或迁移。不要把它提交到 Git、贴到聊天窗口、上传到公共工单或放进 Web 目录。

## 9. Keeper 页面

Keeper 页面用于账号维护监控和批量操作。

### 9.1 演练扫描

点击 `演练扫描` 会按当前 Keeper 策略检查账号，并生成维护建议。演练不会改动 CPA 远端数据。

结果表会显示：

- 账号。
- 建议动作，例如保留、删除、禁用、启用、刷新、刷新候选、跳过或异常。
- 额度。
- 状态。
- 原因。

建议先执行演练，确认结果合理后再执行真实维护。

### 9.2 执行维护

点击 `执行维护` 会按 Keeper 策略对账号执行实际维护。执行完成后页面会尝试刷新账号列表。

维护可能包含：

- 删除不可用账号配置。
- 禁用低额度或异常账号。
- 启用符合条件的账号。
- 刷新临期证书。

如果刷新证书失败，该账号会被记录为 OAuth 页面中的失效候选来源之一。

### 9.3 选中账号直接操作

Keeper 页面下方仍显示账号表格。先勾选账号，再使用：

- `设置禁用 (N)`：禁用选中账号。
- `刷新证书 (N)`：刷新选中账号证书。
- `删除证书 (N)`：删除选中账号证书或账号配置。

删除操作不可逆，页面会先弹确认框。

### 9.4 重复证书筛选

点击 `筛选重复证书` 会按账号信息检测重复授权文件，并给出建议删除项。

操作步骤：

1. 点击 `筛选重复证书`。
2. 检查每组重复证书中页面建议保留和建议删除的项目。
3. 按需调整勾选。
4. 点击 `删除选中重复证书 (N)`。

删除重复授权文件不可逆，建议先下载全量备份。

## 10. OAuth 页面

OAuth 页面用于辅助失效 Codex 账号重新完成 CPA Codex OAuth 登录。

### 10.1 失效账号来源

页面会把以下账号作为失效候选：

- 额度查询后状态为 `异常` 的账号。
- Keeper 刷新证书失败且未成功写回的账号。
- 在 `批量导入失效邮箱` 中导入并匹配到的账号。

未经过额度查询或 Keeper 刷新的账号，默认不会被单独判断为失效账号。账号被禁用、缺少 refresh token 或证书过期，也不会单独作为 OAuth 失效账号判定条件。

### 10.2 导入 Hotmail 账号池

在 `Hotmail 账号池` 区域导入账号。格式为：

```text
账号----密码----ID----Token
alice@hotmail.com----password----client-id----refresh-token
```

可以一次粘贴多行。

Hotmail 邮箱、Client ID、密码和 refresh token 会保存在浏览器本地配置中，方便刷新后继续使用 OAuth 辅助和扩展自动登录。当前 Hotmail 密码和 refresh token 不经过 Web 端运行时密钥加密，请只在可信本机或可信内网浏览器 profile 中导入。

持久化规则：

- 邮箱：保存。
- Client ID：保存。
- 密码：保存到当前浏览器本地配置。
- refresh token：保存到当前浏览器本地配置，用于后续验证码读取和 token 刷新。
- 最近验证码：不长期保存，只用于当前 OAuth 会话展示。

### 10.3 手动 OAuth 重登流程

1. 进入 `OAuth` 页面。
2. 确认已经加载账号列表。
3. 在 `失效账号` 中选择目标账号。
4. 在 `Hotmail 账号池` 中导入并选择同邮箱 Hotmail 账号。
5. 点击 `发起 OAuth登录`。
6. 页面会在新标签页打开 OpenAI OAuth 登录链接。
7. 在 OpenAI 页面请求邮箱验证码后，回到本项目点击 `获取 Hotmail 验证码`。
8. 复制页面展示的验证码，填入 OpenAI 登录页。
9. OpenAI 授权完成后，把跳转得到的 OAuth 回调 URL 粘贴到 `OAuth 回调 URL`。
10. 点击 `提交回调 URL`。
11. 点击 `检查登录状态`。

如果 CPA 返回 OAuth 成功，页面会刷新账号列表，并对目标账号做一次额度查询。额度查询不再是 `异常` 时，该账号会恢复为正常；如果仍为 `异常`，会继续保留在失效账号列表中。

### 10.4 OAuth 批量队列

OAuth 页面顶部提供批量队列：

- `全部失效账号生成队列`：把当前识别出的全部失效账号入队。
- `勾选账号生成队列`：只把账号表格中勾选的账号入队。
- `当前筛选结果生成队列`：按账号表格当前筛选结果入队。
- `清空队列`：清除本地 OAuth 批量队列。

队列表只展示安全摘要，包括邮箱、Hotmail 匹配邮箱、任务状态、尝试次数、最近错误、OAuth 后验状态和更新时间。

`callback 已提交` 只表示扩展捕获并提交了 OAuth callback URL，不等于登录成功。`OAuth success` 表示 CPA 的 OAuth 登录状态检查已经返回成功。

## 11. Chrome / Edge 自动登录扩展

仓库提供可选扩展：

```text
browser-extension/codex-oauth-auto-login/
```

### 11.1 安装扩展

1. 打开 `chrome://extensions` 或 `edge://extensions`。
2. 开启开发者模式。
3. 点击 `加载已解压的扩展`。
4. 选择 `browser-extension/codex-oauth-auto-login` 目录。

### 11.2 使用扩展

1. 先打开本项目页面，并进入左侧 `OAuth` 页面。
2. 如果 Web 运行在远程地址，在扩展侧边栏 `平台绑定` 中填写 `Web 地址`，例如 `http://192.168.1.10:5173`，保存时浏览器会请求该 origin 的可选访问权限。
3. 如果需要持久化扩展侧访问密码，填写 `访问密码` 并勾选 `保存密码`。输入框右侧小眼睛用于切换当前输入明文/隐藏；保存后不会回显已保存密码。
4. 保留这个 Web 页面标签页。未配置平台地址时，地址应为 `http://localhost/...` 或 `http://127.0.0.1/...`；配置后，扩展会匹配绑定的远程 Web origin 和 base path。
5. 点击浏览器工具栏里的 `CPA Codex OAuth` 扩展图标，打开侧边栏。
6. 点击 `读取账号池`，让扩展从 Web 页面读取失效账号、Hotmail 账号池和队列状态。侧边栏只显示邮箱、状态、Client ID 和错误摘要，不展示 Hotmail 密码、refresh token 或验证码。
7. 可在本地 OAuth 页面或扩展侧边栏生成队列。
8. 点击 `开始全部` 或相应启动按钮运行批量登录。
9. 运行期间保持 OpenAI 授权页可见，便于扩展填写邮箱、验证码和授权确认。

扩展遇到 CAPTCHA、MFA、手机号验证、安全检查、Passkey 或无法识别的页面时，会暂停或标记需要人工处理。

平台绑定密码规则：

- 勾选 `保存密码` 并输入新密码：加密保存新密码到扩展本地存储。
- 勾选 `保存密码` 但输入框留空：保留已有已保存密码。
- 取消勾选 `保存密码` 后保存：删除已保存密码。
- 该密码目前只是扩展侧持久化配置项；如果 Web 需要强制校验，还需要配套 Web 端鉴权逻辑。

### 11.3 扩展安全边界

- 扩展默认只用于本地 CPA 页面和 OpenAI 登录相关 host；远程 Web 地址通过 `optional_host_permissions` 按 origin 请求权限，不声明 `<all_urls>`。
- 扩展会清理 OpenAI 会话，降低不同账号 OAuth 流程串号风险。
- 扩展侧边栏会显示必要的 Hotmail 元数据，不展示 Hotmail 密码、refresh token 或验证码。
- 扩展本地存储只保存平台绑定地址、可选的扩展访问密码和自动化参数；不会保存 CPA 管理密钥、Hotmail 密码、Hotmail refresh token 或验证码。
- 页面桥接只接受同窗口同源消息；扩展只会向匹配的 Web tab 注入 `content-cpa.js` 并读取桥接状态。
- OAuth 回调 URL 支持本地 `localhost` / `127.0.0.1` 回调地址；配置远程 Web 后，也支持绑定 origin 下的 `/auth/callback`、`/codex/callback`，以及同路径位于配置 base path 下的回调 URL。回调 URL 必须包含 `state` 和 `code` 或 `error`。
- 远程 HTTP Web 只建议放在可信内网中使用；如果跨公网访问，建议在 Web 前面增加 HTTPS 反向代理。

## 12. 本地数据与缓存

浏览器本地存储分两类：

- `localStorage`：保存 CPA 地址、查询设置、Keeper 设置、优先级分组顺序、优先级区间、界面语言/主题和 OAuth 配置等。管理密钥会以加密对象保存；Hotmail 密码和 refresh token 当前随 OAuth 配置保存在当前浏览器 profile 中。
- `IndexedDB`：保存账号列表缓存和额度快照。

缓存用途：

- 页面刷新后回填最近一次账号列表。
- 查询失败或超时时保留上一次成功额度快照。
- 自动登录失败时，在已有缓存的情况下尽量保留最近可用视图。

清理方式：

1. 点击右上角设置入口。
2. 找到 `本地数据`。
3. 点击 `清空本地缓存`。

清空后会回到登录页。

如需迁移本地敏感配置，可在设置面板点击 `导出敏感配置`。导出的 JSON 是明文，适合人工备份或迁移到另一台可信机器；不应提交到仓库、聊天窗口或公共工单系统。

本地加密边界：

- 加密目的是避免管理密钥以明文长期留在浏览器存储里。
- Hotmail 密码和 refresh token 当前不属于 Web 端运行时密钥加密范围，应只在可信浏览器环境中导入和使用。
- 扩展侧的 `访问密码` 使用扩展本地密钥加密保存在 `chrome.storage.local`，不会回显到侧边栏。
- 运行中的 Web 页面可以在用户确认后导出明文，因此它不是多用户权限隔离系统。
- 保护浏览器 profile、操作系统账号和下载目录仍然很重要。
- 如果运行时密钥丢失，旧加密管理密钥无法恢复，只能重新登录，或从此前导出的敏感配置中手工恢复。

## 13. 常见问题

### 13.1 登录后按钮不可用

检查：

- CPA 管理地址和管理密钥是否都已填写。
- 是否仍处于加载、查询、下载、同步或 Keeper 执行中。
- 当前操作是否需要先加载账号列表或勾选账号。

### 13.2 加载账号失败

检查：

- CPA 服务是否运行。
- 管理地址是否可访问。
- 管理密钥是否正确。
- CPA 或反向代理是否允许当前 Web 页面跨域访问。
- 浏览器控制台是否有 CORS、401、403 或网络超时错误。

### 13.3 额度查询失败或大量异常

可能原因：

- 账号 refresh token 已失效。
- CPA `api-call` 代理不可用。
- OpenAI 相关接口网络异常。
- 管理密钥权限不足。
- 查询并发过高导致请求超时或被限流。

处理建议：

- 降低设置中的并发数。
- 对少量账号重新查询确认。
- 使用 Keeper 或 OAuth 页面处理失效账号。
- 检查 CPA 服务端日志。

### 13.4 同步优先级前一直提示备份

页面只有在本轮生成优先级草稿后执行过全量下载，才认为已有新备份。只下载选中账号不能替代同步前全量备份。

可选择：

- 先下载全量备份再同步。
- 明确确认风险后直接同步。

### 13.5 OAuth 页面没有失效账号

检查：

- 是否已经加载账号列表。
- 是否对账号执行过额度查询。
- 是否有状态为 `异常` 的账号。
- 是否有 Keeper 刷新证书失败的账号。
- 是否需要在 `批量导入失效邮箱` 中手动导入邮箱。

### 13.6 Hotmail 验证码获取失败

检查：

- Hotmail 导入格式是否正确。
- Hotmail 邮箱是否与失效账号邮箱一致。
- Client ID 和 refresh token 是否有效。
- CPA `api-call` 是否能代理 Microsoft token、Graph 或 Outlook 邮件接口。
- OpenAI 邮件是否已经到达，且不是旧验证码。

### 13.7 扩展读取不到账号池或队列

检查：

- CPA Codex Tools Web 页面是否已经打开。
- 页面是否停留在 `OAuth` 页面。
- 如果未配置扩展 `平台绑定`，页面地址是否为 `http://localhost/...` 或 `http://127.0.0.1/...`。
- 如果 Web 部署在远程地址，扩展侧边栏中的 `Web 地址` 是否与当前页面 origin 和 base path 一致，并且浏览器是否授予了该地址权限。
- 是否已经点击扩展侧边栏的 `读取账号池`。
- 浏览器是否禁用了扩展的 Web 页面或 OpenAI host 权限。

### 13.8 刷新页面后 Hotmail refresh token 不见了

当前实现会把 Hotmail refresh token 随 OAuth 配置保存在当前浏览器本地。如果刷新后看不到，优先检查是否清空过本地缓存、是否换了浏览器 profile、是否导入数据时该列为空，或是否被新导入的同账号记录覆盖。

### 13.9 刷新后管理密钥无法解密

常见原因：

- Docker volume 被删除，导致 `/runtime/vault-secret` 重新生成。
- 本地 `web/.runtime/vault-secret` 被删除。
- 使用了不同浏览器 profile 或不同部署环境打开旧缓存。

处理方式：

- 如果还保留旧 runtime secret，恢复它后重新启动 Web。
- 如果曾经导出过敏感配置，用导出文件人工恢复。
- 如果无法恢复密钥，只能清空本地缓存后重新登录和重新导入 Hotmail 账号池。

### 13.10 导出的敏感配置如何迁移

建议流程：

1. 在旧环境设置面板点击 `导出敏感配置`。
2. 把 JSON 文件通过可信渠道移动到新机器。
3. 在新环境手动填入 CPA 地址和管理密钥，并重新导入 Hotmail 账号池。
4. 确认新环境运行正常后，删除临时明文 JSON。

当前导出功能负责生成明文备份文件，不会自动导入并覆盖本地配置。

## 15. 术语速查

- CPA：承载 Codex 账号配置和 Management API 的服务端。
- 管理密钥：访问 CPA Management API 的密钥。
- 运行时加密密钥：浏览器端用于加密本地敏感配置的密钥，Docker 下保存在 `/runtime/vault-secret`。
- 敏感配置导出：设置面板下载的明文 JSON，包含管理密钥、Hotmail 密码和 refresh token。
- auth index：CPA 中账号配置的索引标识。
- 5h / 7d 额度：Codex 在不同时间窗口内的剩余额度。
- 额度更新时间：界面中显示的下一次额度刷新时间。
- 本地草稿：尚未同步到 CPA 的优先级修改。
- Keeper：账号维护逻辑，用于检查、禁用、刷新或删除账号配置。
- OAuth 回调 URL：OpenAI 授权完成后跳转到本地 callback 的 URL，需要提交给 CPA 完成登录状态写回。
