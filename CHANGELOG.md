# Changelog

本文件记录每个公开版本的用户可感知变化，便于后续发布、回滚和问题追溯。

## [Unreleased]

## [0.2.0] - 2026-05-12

### 更新项

- Web 端支持对管理密钥进行运行时密钥加密保存，并提供敏感配置导出能力。
- Codex OAuth Hotmail 账号池支持保存密码和 refresh token，刷新页面后可继续用于验证码读取和 OAuth 辅助。
- 浏览器扩展增加远程 Web 平台地址绑定，可连接部署在服务端的 Web 页面。
- 浏览器扩展增加平台访问密码持久化配置，支持勾选保存密码和按需清除保存密码。
- 浏览器扩展密码输入框增加明文/密文显示切换。
- README、使用手册和扩展说明补充远程部署、HTTP 使用边界、权限申请和敏感数据保存范围。

### 优化项

- Web 端敏感数据说明收敛为明确边界：当前加密保存 CPA 管理密钥，Hotmail 密码和 refresh token 保存在浏览器本地配置中。
- 扩展远程平台权限改为按绑定地址 origin 请求 `optional_host_permissions`，避免使用全量站点权限。
- 扩展侧边栏只展示账号摘要和任务状态，不暴露 Hotmail 密码、refresh token 或验证码。
- 扩展保存密码逻辑调整为显式勾选，空密码保存时保留已有密码，取消勾选时清除已保存密码。
- OAuth bridge 与扩展 README 同步实际动作清单，降低后续对接和排障成本。

## [0.1.6] - 2026-05-10

### 新增

- 增加 Codex OAuth 批量自动登录浏览器扩展，支持队列生成、暂停/继续、任务状态和错误日志。
- 增加 Hotmail 账号池导入、验证码读取和 OAuth callback 提交桥接。
- 增加任务间隔、点击超时、跳过等待和任务超时等自动化参数。

### 优化

- OpenAI 登录流程支持一次性验证码登录、中文页面识别、受信任点击回退和 OpenAI 会话清理。
- 账号停用 `account_deactivated` 页面会自动标记当前任务失败并跳过。
- 获取邮箱验证码前增加额外等待，降低验证码尚未入箱导致的失败。
- 增强 OAuth callback、管理接口地址和本地缓存的校验与脱敏。

## [0.1.5] - 2026-05-05

### 优化

- 完善批量生成优先级逻辑，按当前账号列表排序在分组内从高到低生成本地草稿。
- 批量生成优先级时只影响当前筛选列表中的勾选分组，避免覆盖当前筛选外账号。
- README 增加批量生成优先级的规则说明和推荐使用流程。

## [0.1.4] - 2026-05-05

### 优化

- Keeper 删除证书前增加二次确认，取消后不会调用删除接口或进入执行状态。
- 修正额度刷新时间的数据语义，新增真实重置时间字段用于排序，展示仍保留短时间标签。
- 清理未使用的 DetailPanel 组件、测试和相关样式，减少维护成本。
- 新增 `typecheck`、`test:run` 脚本和 Web CI，发布前自动覆盖类型检查、测试与构建。
- 同步 README 中 Keeper 自动刷新和本地缓存说明。

## [0.1.3] - 2026-05-04

### 新增

- 增加 Keeper 功能，支持在账号列表中指定账号后直接禁用、刷新证书和删除证书。

### 调整

- 完善额度获取流程，完成额度获取后会把账号列表与额度快照持久化到本地缓存。

## [0.1.2] - 2026-05-03

- 项目入口收敛为 Web-only，本地使用通过浏览器或 Docker Compose 启动。
- 移除 Python CLI、桌面壳层、桌面构建脚本和相关测试。
- 前端工程目录从 `desktop/` 调整为 `web/`。
